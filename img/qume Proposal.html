About the Project?

Enabling accurate magnetic field measurements on Cubesats using the BeagleBone platform

Magnetic field measurements play an important role in space weather and engineering applications of Earth-orbiting satellites, such as attitude determination, momentum management, and scientific instrument pointing. Unless built specifically for high accuracy magnetic measurements (an expensive process), satellites usually come with significant magnetic sources of errors that severely degrade the accuracy with which the Earth’s field can be measured. This GSoC project aims to implement algorithms that enable low-cost high quality magnetic field measurements on smaller spacecraft without booms using the BeagleBone computing platform. The project should serve as a demonstrator for applying current state-of-the-art sensor fusion algorithms using DIY of-the-shelf components, as well as, providing the Cubesat community with a software framework for easily applying advanced sensor fusion algorithms to their own affordable yet accurate satellite projects.

The first phase of the Orbital Imaging Cubesat project, includes signal acquisition from external IMU sensor modules, implementing algorithms for calculating compensation parameters and processing magnetometer data with error compensation algorithms on the PRUs. In addition, the processed data should be made available as sysfs entries for easier future access to data using in the project API.This will be achieved by writing a kernel driver to control data processing on the PRUs.

Signal acquisition will be achieved using the two Programmable Realtime Units (PRUs) available on the BeagleBone chipset connecting them with adequate IMU sensor boards via one of the available GPIO interfaces. The most likely candidate for the sensor module is the Invensense MPU-9250 with its combined accelerometer/gyro and an integrated Asahi Kasei AK8975A magnetometer. This module has already been adapted in the DIY community, with a BeagleBoard cape developed for it [1], and drivers available for a similar model which lacks the magnetometer [2].
The first part would consist of connecting the hardware modules and acquiring raw data from the sensors. The possible methods of connecting the modules is either through the dedicated BeagleBone interfaces (I2C or more probably SPI) using the available user space or kernel drivers, accessing the interfaces directly through the PRU or bit-banging the protocols on the PRUs themselves. The method will be selected based on testing and feedback from mentors during the bonding period.

In the interest of efficient correction of errors in magnetometer data, possibilities of offloading some or all processing of the acquired data on the PRUs themselves would be investigated, with a focus on applying fixed point arithmetic algorithms to correct the raw data. Due to the limited capabilities of the PRUs architecture and instruction set, it may be required that compute intensive algorithms be implemented in the main ARM core.
Specifically, we can split the algorithms in two categories, real-time algorithms for data correction using already computed correction parameters and algorithm for computing the correction parameters. Magnetometer data correction can be done using pre-computed correction parameters and equations described in [1], [4] and [8]. Pending further testing, these equations should be able to run in part or fully on the PRUs, thus offloading most of the real-time data processing from the main ARM CPU. Raw data readouts should be fed to the PRUs for real time processing, ideally in fixed point arithmetic. The potential limiting factors for this design are the uncertainty in the time spent fetching data outside of the PRU memory space and the time difference between data acquisition from the sensors and data correction in the PRU. Based on current testing, these algorithms should fit in the PRUs instruction space and further optimizations will be done based on mentor feedback.
The second algorithm set for correction parameter computation will most probably be done on the ARM core due to the complexity of minimizing the least squares cost function to find the optimal parameters as described in [8] and other related papers. These algorithms will not lend themselves to porting to the PRU and are themselves not meant to be used real-time, but periodically with a body of collected sensory data.
Control of the processing done on the PRUs will be achieved by implementing a kernel driver for the PRU and processed data will be made available through sysfs entries. The kernel driver will be developed based on current kernel and user-space drivers for the PRU, with the intent of automating often used real-time data correction code. The specifics of the sysfs modules to be exposed and the kernel driver details will be agreed upon during the bonding period. 

 

This concludes the work proposed to be done during this GSoC project. After the phase I implementations, additions to the project which should be considered include further testing of the magnetometer data correction systems with advanced testing equipment as described in [4] and subsequent research papers, adding support for other subsystems in subsequent phases of the project and building a higher level API for easy manipulation of data and general control of the BeagleBone based CubeSat. Details on the possible expansions are outlined below.


The complete system with sensors and applied algorithms should be tested with an apparatus with multiple magnetometers. The apparatus should consist of a coil and magnet to simulate variant and invariant magnetic sources as described in [4], which broadly corresponds to the instruments being tested in the NASA article [3]. Since the information regarding the newest Dellingr research is not complete, testing should be done with the current proposed system and available research, with more refinement possible pending information from NASA. For performance measuring purposes, the goal is to have visible improvements in the magnetometer data compensation for time invariant sources on par with the data presented in [4] and compensation for variant sources should strive to be on par with that of invariant sources after applying the mentioned algorithms. The final testing methodology will be expanded through consultation with the project mentors.


Further consideration for the project consist of implementing algorithms for improving magnetometer readouts using data from multiple IMU sensors and is based on a number of NASA research studies for future measurement equipment on board small satellites. These novel sensors systems are developed for the Dellingr nano satellite [3] and adapting this system using a BeagleBone board would provide the Cubesat community with the framework and tools necessary to deploy such an advanced system with ease in the near future. Unfortunately, the papers for this particular system are yet to be published, pending the launch of Dellingr, though it is possible to replicate the work done on Dellingr to a high degree following related papers and published works.

The improvements mentioned should be implemented either after the completion or as a stretch goal for this GSoC project, depending on available time and at the mentors' discretion. Due to the time and resources required to implement them and the duration of Google Summer of Code, they are not listed in the project main timeline, but would further strengthen the BeagleBone as a capable and versatile CubeSat platform.
